# 离线 PMRID GPU RAW 降噪设计

日期：2026-07-14  
状态：已由用户逐段确认

## 1. 目标与范围

在现有 Sony A7C II ARW 到可编辑 CFA DNG 的可靠流程上，新增完全离线的 GPU RAW 降噪引擎。目标设备是配备 8 GB 显存的 RTX 3070 Ti Laptop。应用默认全自动处理任意长度的批量队列，同时保留可展开的高级参数、CPU 回退和可打包的 Windows GUI。

本阶段采用 PMRID 的 Bayer RAW 网络和官方 PyTorch 预训练权重，转换为 FP16 ONNX 后随安装包发布。PMRID 代码及权重来源于官方 Apache-2.0 项目。模型原始训练域主要是手机传感器，因此产品中必须标为“通用 RAW GPU 模型”，不得描述为 A7C II 专用模型。本阶段不训练或宣称拥有相机专用权重。

以下能力不属于本阶段：生成式细节、RGB 域降噪、自动曝光、白平衡、锐化、调色、云端推理和运行时下载模型。

## 2. 技术路线

现有 ARW 解码、Bayer 归一化、CFA 排列、DNG 写入和校验路径保持不变。新增 `GpuRawDenoiser`，通过统一接口接收四通道归一化 Bayer 与噪声参数，返回同尺寸四通道 Bayer。

推理运行时采用 ONNX Runtime CUDA Execution Provider。Windows 离线包包含：

- FP16 ONNX 模型；
- ONNX Runtime GPU 运行库；
- 与固定 ONNX Runtime 版本匹配的 CUDA 和 cuDNN 动态库；
- 模型与第三方组件的许可和来源信息；
- 模型清单及 SHA-256 校验值。

首个 GPU 发布基线固定为 `onnxruntime-gpu 1.23.2`、CUDA 12.x 和 cuDNN 9.x。构建锁文件继续固定实际打包的每个 NVIDIA wheel 与 DLL 版本；升级任一运行时组件必须重新完成 GPU 自检、整图性能测试和离线干净环境冒烟测试。

应用不要求安装 Python、CUDA Toolkit 或联网下载权重。NVIDIA 显卡驱动仍是系统前置条件。启动检测必须执行一次小尺寸真实推理，不能仅凭设备名称或 provider 列表判断 GPU 可用。

## 3. 组件边界

### 3.1 模型清单

模型清单记录名称、语义版本、文件名、SHA-256、许可、来源提交、输入输出名称、通道排列、数值范围、精度、推荐分块和兼容运行时。加载前校验清单和文件哈希；不匹配时拒绝 GPU 加载并给出可操作错误。

### 3.2 GPU 推理引擎

`GpuRawDenoiser` 负责创建 ONNX 会话、选择 CUDA provider、FP16 张量转换、噪声条件输入、分块调度和结果检查。GUI、队列和 DNG 模块只依赖统一降噪接口，不依赖 PMRID 的内部网络结构。

### 3.3 自动参数估计

自动控制器综合 ISO、曝光时间、RAW 暗部统计和现有盲噪声估计，产生每张照片独立的降噪配置。估计置信度不足时降低强度，默认保留少量自然颗粒。自动控制器不得改变曝光、白平衡、色彩矩阵或锐度。

### 3.4 CPU 回退

现有 CPU 引擎保留为明确的兼容路径。GPU 完全不可用、模型损坏或所有安全分块均失败时，任务才回退 CPU。界面和任务记录必须显示实际使用的引擎，不能把 CPU 处理显示为 GPU。

## 4. 数据流

```text
ARW（只读）
  -> rawpy/dnglab 解码及元数据读取
  -> 按 CFA 位置做黑白电平归一化
  -> 打包 R/G1/G2/B 四通道 Bayer
  -> 自动噪声估计与强度选择
  -> ONNX Runtime CUDA 分块推理
       -> CUDA OOM：逐级缩小分块并重试
       -> GPU 不可用：明确记录并回退 CPU
  -> 细节保护、结果混合和范围检查
  -> 恢复原始黑白电平及 CFA 排列
  -> 写入临时 CFA DNG
  -> 结构和元数据校验
  -> 原子发布最终 DNG
```

模型输出按强度与原始归一化 RAW 混合，使高级强度滑块具有稳定、可解释的效果。所有输出在恢复传感器编码前检查有限值和合法范围。

## 5. 分块、显存与性能

33 MP 输入使用带重叠的分块推理和权重窗融合，禁止硬拼接。默认分块由启动自检和可用显存估算选择。CUDA OOM 时清理会话可回收缓存，并按预定义尺寸逐级缩块；每个尺寸只重试一次，防止无限循环。

任务记录实际 provider、模型版本、精度、分块尺寸、推理耗时和峰值显存。RTX 3070 Ti Laptop 普通模式的目标是平均不超过 45 秒/张。性能验收必须使用完整 A7C II 33 MP ARW，而非只报告小块推理速度。

## 6. 自动模式与高级参数

默认界面只显示“自动”。高级区域展开后提供：

- 降噪强度：控制模型结果与原始 RAW 的混合比例；
- 彩色噪点：控制 R/B 与两个绿色位置之间的噪声一致性约束；
- 细节保护：降低结构边缘和高频纹理区域的处理权重；
- 伪影抑制：控制异常色点、坏点和分块边缘的保守修复。

每张任务在加入队列时保存参数快照。修改全局设置只影响之后加入或明确重新应用设置的任务。

## 7. GUI 与目录工作流

主界面显示实际 GPU 名称或 CPU 回退、CUDA provider 状态、模型名称和版本、当前分块尺寸、显存占用、处理阶段及单张耗时。设置页提供“GPU 自检”，错误信息可复制。

设置页新增：

- 默认导入目录；
- 默认导出目录；
- 导出策略：“固定导出目录”或“每个源目录下的 `DNG_Denoised`”。

文件和文件夹选择器默认从已保存的导入目录打开，并记住最近使用位置。主界面提供“打开导出目录”按钮。单个已完成任务提供“定位文件”，在 Windows 资源管理器中选中对应 DNG。

任务加入队列时保存解析后的输出路径。之后修改目录设置不得改变现有任务的输出位置。输出冲突继续使用自动编号且绝不静默覆盖。

## 8. 错误处理与恢复

- GPU 自检失败：显示驱动、运行库或 provider 的具体错误，允许 CPU 模式继续；
- CUDA OOM：逐级缩块，全部失败后回退 CPU 并记录原因；
- 模型或清单损坏：拒绝模型加载，不执行未经校验的权重；
- 输出目录不可写或空间不足：处理前暂停相关任务，不开始 DNG 写入；
- 单张 ARW 失败：仅标记该任务，继续后续队列；
- 程序中断：依赖 SQLite 状态恢复，运行中任务回到可重试队列；
- DNG 写入或校验失败：删除临时文件，不发布不完整输出；
- 源 ARW：所有路径只读，处理前后内容哈希保持一致。

## 9. 测试与验收

### 9.1 单元和集成测试

- 模型清单解析、版本和 SHA-256 校验；
- CUDA provider 选择和实际自检；
- FP16 输入输出的形状、范围和有限值；
- 分块覆盖、权重融合和接缝阈值；
- OOM 缩块顺序、重试上限和 CPU 回退；
- 自动参数在不同 ISO 和置信度下的边界；
- 高级滑块对结果的单调、有限影响；
- 目录设置持久化、任务路径快照、打开目录和定位文件；
- 模型损坏、磁盘不足、不可写目录和进程中断。

### 9.2 真实文件验收

- 使用 A7C II 实拍 ARW 完成 GPU Bayer 降噪和 CFA DNG 导出；
- Adobe Camera Raw/Lightroom 与像素蛋糕可导入并继续调整白平衡、曝光和 RAW 参数；
- 检查暗部色斑、细发、布料、星点、边缘光晕、坏点和分块接缝；
- 对比原图、现有 CPU 基线和 GPU 输出，不允许整体曝光或色彩发生非预期改变；
- 在至少 150 张队列中验证暂停、恢复、失败重试、重启恢复和单张失败隔离；
- 在 RTX 3070 Ti Laptop 上记录驱动、provider、模型、分块、显存和整张处理时间。

## 10. 发布约束

发布物是完整离线 Windows 包。构建固定依赖版本并生成第三方组件清单、模型来源和哈希。安装包启动时不联网，也不包含自动下载逻辑。包体增大是已接受的取舍。

PMRID 初始权重作为通用 RAW 基线发布，其跨传感器限制必须在界面和 README 中明确。后续 A7C II 专用或多相机 NAFNet 权重可以复用本设计的模型清单、GPU 引擎、分块调度和 GUI，不需要改动 DNG 主流程。
