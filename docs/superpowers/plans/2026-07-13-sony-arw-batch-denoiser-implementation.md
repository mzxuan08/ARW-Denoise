# Sony ARW 批量降噪软件实施计划

日期：2026-07-13  
依据：`docs/superpowers/specs/2026-07-13-sony-arw-batch-denoiser-design.md`

## 实施原则

采用纵向切片推进，每个阶段都产生可运行、可测试的结果。首先验证 CFA DNG 兼容性，再投入模型训练和复杂 GUI。源 ARW 始终只读；所有输出使用临时文件、校验和原子改名。

## 阶段 1：项目与测试基础

### 任务 1.1 项目骨架

创建 `pyproject.toml`、`src/arw_denoise/`、`tests/`、许可证、README 和开发命令。Python 最低版本定为 3.11。核心包不得在导入时强制加载 PySide6、rawpy 或 CUDA，以便测试和 CPU 环境运行。

验收：可执行 `python -m arw_denoise --help`；基础单元测试通过。

### 任务 1.2 配置与结构化日志

实现应用目录、输出命名、模型元数据和日志脱敏。使用标准库数据结构保存设置，后续 GUI 复用。

验收：路径冲突自动编号；日志默认不记录完整源路径。

## 阶段 2：格式兼容性闸门

### 任务 2.1 ARW 探测接口

实现 `RawMetadata`、`RawFrame` 和 `RawDecoder` 接口；创建 rawpy/LibRaw 适配器，提取 CFA、尺寸、black/white level、位深与基础 EXIF。拒绝未知 CFA 或缺失关键数值范围。

验收：使用模拟解码器完成测试；获得真实 A7C II 样本后补充集成测试。

### 任务 2.2 dnglab 适配层

实现 dnglab 可执行文件发现、版本探测、超时、进程错误解析和临时输出。第一步先做未经降噪的 ARW → DNG 往返，用来验证 Adobe 与像素蛋糕接受的 DNG 编码。

验收：模拟进程测试通过；真实样本能生成 DNG；失败不留下最终文件。

### 任务 2.3 DNG 结构快照与兼容性清单

记录原始元数据快照和输出结构，建立 Adobe、像素蛋糕人工验收表。只有 CFA、范围和关键元数据验证通过，才进入正式像素替换写入。

验收：形成可重复的兼容性报告；闸门状态可在 CLI 查询。

## 阶段 3：任务系统与 CPU 端到端版本

### 任务 3.1 SQLite 队列

实现 schema、迁移、事务状态转换、恢复、重试、暂停、取消和每图参数快照。运行态任务在异常重启后回到可重试队列。

验收：状态机非法转换被拒绝；150 项模拟队列的恢复测试通过。

### 任务 3.2 Bayer 数值管线

实现每通道 black level 归一化、RGGB 打包/解包、重叠分块、权重融合、范围恢复和有限值检查。

验收：打包往返无损；恒定图与边界图不产生接缝、NaN 或越界。

### 任务 3.3 CPU 降噪基线

实现噪声参数估计接口、方差稳定化和可替换的 BM3D/小波后端。若可选依赖缺失，给出明确错误而不是静默使用劣质算法。

验收：合成 Poisson-Gaussian 噪声图的误差降低；结构边缘退化在阈值内。

### 任务 3.4 降噪 CFA 写回 DNG

扩展 dng_worker，使其写入处理后的 CFA 数据并映射关键 DNG 标签。写入后重新读取验证并原子发布。

验收：Adobe/像素蛋糕兼容；源文件哈希不变；故障注入不发布半成品。

## 阶段 4：GPU 模型

### 任务 4.1 推理接口与分块调度

定义统一模型接口、模型清单、哈希和运行时版本；实现 ONNX Runtime CUDA provider 探测、动态分块、OOM 缩块重试和 CPU 回退。

验收：模拟 OOM 路径可重复；权重哈希错误时拒绝加载。

### 任务 4.2 普通模式模型

实现 Bayer NAFNet/PMRID 类网络和训练配置。使用许可证兼容的多相机 RAW 数据与合成噪声训练，固定验证集并记录指标。

验收：固定数据集上不弱于 CPU 基线，真实 A7C II 图像无明显接缝、色偏和蜡感。

### 任务 4.3 极暗模式模型

增加严重欠曝数据与高噪声采样，限制生成式纹理。采用更大模型或多阶段推理，但保持 CFA 输出。

验收：欠曝提亮 3–5 档样片的色斑和亮度噪声显著下降，不制造可见虚假细节。

## 阶段 5：桌面 GUI

### 任务 5.1 主窗口与队列模型

实现三栏主窗口、固定任务栏、拖放导入、筛选、多选和状态更新。所有耗时操作在工作进程执行。

验收：150 项队列滚动流畅，处理期间 UI 可响应。

### 任务 5.2 设置、预设与恢复

实现自动/极暗/CPU 模式、高级滑块、输出目录、预设以及启动恢复提示。

验收：任务保存参数快照；修改全局设置不改变已运行任务。

### 任务 5.3 局部对比预览

实现当前视口分块预览、参数防抖、过时任务取消、同位置切换和 1:1 检查。

验收：基准机器首次预览目标 5 秒内出现，不处理整张 33 MP 图像。

## 阶段 6：稳定性、打包与发布

### 任务 6.1 集成与故障测试

覆盖损坏 ARW、未知机型、磁盘满、模型损坏、CUDA OOM、dng_worker 崩溃和程序强制退出。验证源 ARW 哈希不变。

### 任务 6.2 性能测试

在 RTX 3070 Ti Laptop 上记录驱动、功耗、模型、分块和 RAW 类型。普通模式目标 45 秒/张以内，极暗模式 120 秒/张以内。

### 任务 6.3 Windows 安装包

冻结 Python 应用，捆绑许可证和 dng_worker，执行干净 Windows 用户环境冒烟测试。首版允许未签名，但必须明确信誉提示。

### 任务 6.4 发布材料

提供 README、安装说明、模型/数据来源、第三方许可证、已知限制、兼容机型矩阵和可复现构建命令。

## 当前迭代范围

本次先完成阶段 1、阶段 2 的可测试适配层、阶段 3 的队列与数值/CPU 基线，以及阶段 5 的最小可运行 GUI。真实 A7C II → DNG 与 Adobe/像素蛋糕兼容性需要真实 ARW 样本及本机应用参与，无法用模拟文件替代；这些项目会明确标记为外部验收，而不会伪报通过。
